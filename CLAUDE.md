## Project Overview

This is the **intern-skills** repository — a collection of AI skill definitions for the Autonomous Intern platform (OpenClaw). Skills are Markdown-based prompt templates organized by professional role. There is no application code (no Go, Python, etc.) — the repo is entirely Markdown skill definitions, a JSON manifest, and a release script.

## Repository Structure

- `skills/<role>/<skill-name>/SKILL.md` — Individual skill definitions (trigger rules, instructions, output format)
- `skills_zip/` — Pre-built zip archives per role, generated by `release.sh`
- `manifest.json` — Maps roles to their keywords, skill lists, and zip filenames
- `version.txt` — Single semver string (current version of the skill pack)
- `release.sh` — Bump version, rebuild zips, commit, tag, and optionally push

## Roles

Nine roles defined in `manifest.json`: `ops`, `finance`, `hr`, `pm`, `cs`, `developer`, `marketing`, `designer`, `sales`. Plus a `generic` role installed for all users. Each role has 5 skills.

## SKILL.md Format

Each skill follows a consistent structure:
- **Description** — One-line summary
- **Trigger** — When the skill activates (user phrases/intents)
- **Instructions** — When to activate, process steps, rules
- **Output Format** — Expected response template with emoji markers

## Release Process

```bash
./release.sh patch   # 1.0.0 -> 1.0.1
./release.sh minor   # 1.0.0 -> 1.1.0
./release.sh major   # 1.0.0 -> 2.0.0
./release.sh 2.3.1   # exact version
```

This script: updates `version.txt`, updates `manifest.json` version field, rebuilds all zip files in `skills_zip/`, then prompts to commit, tag, and push. Requires `jq` for clean JSON updates (falls back to `sed`).

## Key Conventions

- When adding a new skill, create `skills/<role>/<skill-name>/SKILL.md` and add the skill name to the role's `skills` array in `manifest.json`.
- When adding a new role, create the skills directory, add the role entry to `manifest.json`, and run `release.sh` to generate the zip.
- Skill names use kebab-case. Role directory names match the role key in `manifest.json`.
- Version in `version.txt` and `manifest.json` must stay in sync (handled by `release.sh`).
